Hexo

hexo学习笔记

安装

详见hexo官网

$ npm install -g hexo-cli

创建博客

  1. 初始化:hexo init <demo> 如果指定 demo，会在目前的文件夹下建立一个名为 demo 的文件夹。

  2. 初始化后，要记得在当前文件夹下运行npm install命令，安装依赖的包。

  3. 创建博文hexo new [layout] <title>,[layout]可以不指定。

  4. 生成静态文件 hexo generate。

  5. 在当前电脑上查看博客hexo server。

  6. 根据提示，在浏览器上输入相应的地址，即可查看。

示例：

$ hexo init my-first-blog   //初始化
$ npm install      //安装依赖包

$ hexo new 'my new post'　　//新建一篇博文
$ hexo generate　　　//生成静态文件
$ hexo server      //启动服务

然后根据提示，到浏览器输入相应地址(如：localhost:4000)，查看刚才创建的内容。

上传到github

  1. 按官网的方式配置github，单击查看配置方法。
  2. 配置过程需要设置SSH keys，单击查看配置方法。
  3. 在github中创建一个repository，记下路径(如https://github.com/username/username.github.io)。
  4. 用npm安装 hexo-deployer-git.
  5. 设置主目录下的_config.yml文件。
  6. 执行命令hexo deploy
    _config.yml文件的主要设置内容如下:
# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://username.github.io

deploy:
  type: git
  repo: https://github.com/username/username.github.io

接下来在终端中执行

$ npm install hexo-deployer-git --save
$ hexo deploy

传输完成后，在浏览器中输入刚才在url中写下的地址访问博客(如：username.github.io)。

错误集锦

下面是一些我在使用Hexo过程中碰到的一些问题

Cannot　read　property　‘offset’　of　null

原因: _config.yml下timezone配置错误。

解决方法: 将timezone配置成时区名称。

例如: timezone:Asia/Shanghai

ENOSPC Error (Linux)

官网上有这个错误的解释,处理过程如下：

运行 npm dedupe,如果这运行这条命令不能解决问题，则输入下面的命令：

echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf && sudo sysctl -p

客户端

  1. 引用脚本： 服务端运行后，会自动在根目录下生成客户端的js文件，客户端可以通过固定路径的方式，设置<script>标签的scr属性值为 /socket.io/socket.io.js，引用客户端脚本。
  2. 建立连接： 通过var socket = io.connect(url)命令，创建与服务器的连接，要注意的是url参数应为包含’http://‘ 在内的完整地址，或者相对路径，或者为空。
  3. 接收数据： 通过socket.on('event',function(data) {……})命令，接收从服务端发送的数据。
  4. 发送数据： 通过socket.emit('event',args)命令，将数据args发送给服务器，其中’event’事件为服务端使用’on’方法创建的事件。

以上内容参考socket.io官方网站。
